Some Rules

  • Data needs to be tidy. This means:
    • Each row is an observation
    • Each variable is a column
    • Data should be in a single dataset

Piping

  • What we just did works, but isnt very elegant
  • Rather than going step by step, its better to put it all together
  • The operator %>% does this
  • Any function with a f(x,y) argument can be written x %>% f(y)

Piping

> cmut <- choc %>%
+   mutate(weighting = (boost + lion)/2) %>%
+   arrange(desc(boost))
Group boost bounty lion mars snickers wispa yorkie weighting
DevSci 1.0 0.40 0.60 0.69 0.32 0.70 0.00 0.800
QSP 0.6 0.28 0.25 0.32 0.60 0.54 0.12 0.425
Cognitive 0.2 0.70 0.40 0.80 0.10 0.62 0.28 0.300

A note

  • Its important to remember that in R, there are many ways of doing things
  • These functions are the most user-friendly for this, but there are other options!
  • Our filtered argument above could instead be written:
> cfil <- choc[choc$boost >= 0.5]
  • Play with some of the options!
